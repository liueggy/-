## 产品概述

**QSM368ZP-WF** 是移远通信推出的新一代 Linux/Android 智能开发套件，基于瑞芯微 RK3568 IoT 芯片。

### 核心特性
- **处理器**: 四核 ARM Cortex-A55 CPU
- **图形处理**: ARM Mali G52 GPU
- **AI算力**: 1 TOPS NPU
- **图像处理**: 8M ISP HDR
- **无线连接**: Wi-Fi 5 + 蓝牙 4.2
- **视频能力**: 支持 4K 视频解码

### 应用场景
- 智慧工厂边缘设备
- 工控领域应用
- 智能视频监控 (NVR/DVR)
- 深度学习算法部署
- 多媒体处理

## 快速开始

### 环境准备

1. **开发环境**
   - Linux 系统（推荐 Ubuntu 18.04/20.04）
   - ARM64 交叉编译工具链
   - ADB (Android Debug Bridge) 工具

2. **连接设备**
   - 通过 USB 连接开发板
   - 确保 ADB 可正常识别设备
   ```bash
   adb devices
   ```

3. **编译与烧录指导**
   - 参考文档: `Linux/Document/Quectel_QSM368ZP-WF&SG368Z系列_Linux&Ubuntu&OpenWrt_编译&烧录指导_V1.2.pdf`

### 基本操作

```bash
# 进入 Shell
adb shell

# 文件传输（从 PC 到设备）
adb push <本地文件/文件夹> <设备路径>

# 文件传输（从设备到 PC）
adb pull <设备路径> <本地路径>

# 查看日志
adb logcat
```

## 功能示例

### GPIO控制

#### 功能说明
控制 GPIO 引脚输出高/低电平，可用于控制 LED、继电器等外设。

#### 快速使用

1. **GPIO 编号转换**
   - QSM368Z GPIO 硬件编号需要转换为 Linux GPIO 编号
   - 转换公式：
     ```
     GPIO0_D3: 0×32 + 3×8 + 3×1 = gpio27
     GPIO2_A1: 2×32 + 0×8 + 1×1 = gpio65
     ```

2. **编译示例代码**
   ```bash
   # 使用 ARM64 交叉编译工具
   aarch64-linux-gnu-gcc gpio0_d3_control.c -o gpio0_d3_control
   ```

3. **运行示例**
   ```bash
   # 上传到设备
   adb push gpio0_d3_control /data/

   # 添加执行权限并运行
   adb shell
   cd /data
   chmod +x gpio0_d3_control
   ./gpio0_d3_control
   ```

#### 示例代码位置
- `examples/gpio/gpio0_d3_control.c`
- `examples/gpio/README.md`

#### 参考文档
- GPIO 配置文档：查看 QSM368Z gpio configuration 文档

### UART串口通信

#### 功能说明
通过 UART 串口实现数据收发，可用于与外部传感器、模块通信。

#### 快速使用

1. **硬件连接**
   - 开发板 UART8 测试点（引脚 22, 24）
   - 使用 USB 转 TTL 模块连接到 PC
   - 参考位置：查看《Quectel_QSM368ZP-WF_用户指导_V1.0》

2. **串口配置**
   - 端口：UART8 (通常为 /dev/ttyS8)
   - 波特率：9600
   - 数据位：8
   - 停止位：1
   - 校验：无

3. **编译示例代码**
   ```bash
   aarch64-linux-gnu-gcc UartRTtest.c -o UartRTtest
   ```

4. **运行示例**
   ```bash
   # 上传程序
   adb push UartRTtest /data/

   # 运行（自动接收并回显数据）
   adb shell
   cd /data
   chmod +x UartRTtest
   ./UartRTtest
   ```

#### 功能特点
- 自动接收串口数据
- 打印接收到的数据到控制台
- 将接收数据原样回传到串口

#### 示例代码位置
- `examples/uart/UartRTtest.c`
- `examples/uart/README.md`

#### 1. YOLOv5 目标检测 🎯

##### 功能说明
使用 YOLOv5 模型进行实时目标检测，可识别人物、车辆等多种物体。

##### 快速使用

```bash
# 1. 上传文件
adb push examples/rknn/rknn_yolov5_demo/ /data/

# 2. 进入目录
adb shell
cd /data/rknn_yolov5_demo/

# 3. 设置依赖库环境
export LD_LIBRARY_PATH=./lib

# 4. 添加执行权限
chmod +x rknn_yolov5_demo

# 5. 运行检测
./rknn_yolov5_demo model/yolov5s_relu.rknn model/bus.jpg

# 6. 获取结果图片
exit  # 退出 adb shell
adb pull /data/rknn_yolov5_demo/out.png ./
```

##### 输入输出
- **输入**: 图片文件（JPG/PNG）
- **输出**: 标注了检测框和标签的图片 `out.png`
- **模型**: YOLOv5s RKNN 格式
- **标签**: 80类 COCO 数据集标签（见 `model/coco_80_labels_list.txt`）

##### 示例位置
- `examples/rknn/rknn_yolov5_demo/`

## 常见问题

### Q1: ADB 无法连接设备？
**解决方案**:
1. 检查 USB 连接是否正常
2. 确认已安装 ADB 驱动
3. 执行 `adb devices` 检查设备状态
4. 尝试 `adb kill-server` 后重新连接

### Q2: 程序运行时提示权限不足？
**解决方案**:
```bash
chmod +x <可执行文件>
# 或者使用 root 权限
adb root
adb remount
```

### Q3: RKNN 示例运行失败？
**解决方案**:
1. 确保已设置环境变量：`export LD_LIBRARY_PATH=./lib`
2. 检查模型文件是否完整
3. 确认输入图片/音频格式正确
4. 查看详细错误日志

### Q4: 如何编译自己的程序？
**解决方案**:
```bash
# 使用 ARM64 交叉编译工具
aarch64-linux-gnu-gcc -o output_file source_file.c

# 链接额外的库
aarch64-linux-gnu-gcc -o output_file source_file.c -L./lib -lrockchip_rknn
```

### Q5: GPIO 控制没有响应？
**解决方案**:
1. 确认 GPIO 编号转换正确
2. 检查引脚是否被其他功能占用
3. 查看内核日志：`dmesg | grep gpio`
4. 确认引脚复用配置正确

### Q6: UART 无法收发数据？
**解决方案**:
1. 检查波特率设置是否一致（推荐 9600）
2. 确认串口设备节点正确（如 /dev/ttyS8）
3. 检查硬件连接：TX → RX, RX → TX
4. 使用 `cat /proc/tty/drivers` 查看可用串口

### Q7: 如何添加自己的 RKNN 模型？
**解决方案**:
1. 使用 RKNN-Toolkit 转换模型
2. 参考示例代码修改输入输出处理
3. 按照 RKNN API 文档编写推理代码
4. 详见 RKNN 用户指导文档

### Q8: 内存不足怎么办？
**解决方案**:
```bash
# 查看内存使用情况
free -h

# 清理缓存
sync; echo 3 > /proc/sys/vm/drop_caches

# 关闭不必要的进程
ps aux | grep <进程名>
kill <PID>
```

### GPIO 转换公式

```
GPIO[组]_[区][编号] = 组×32 + 区×8 + 编号×1

其中:
- 组: 0, 1, 2, 3, 4
- 区: A=0, B=1, C=2, D=3
- 编号: 0-7

示例:
GPIO0_D3 = 0×32 + 3×8 + 3 = 27
GPIO2_A1 = 2×32 + 0×8 + 1 = 65
```
